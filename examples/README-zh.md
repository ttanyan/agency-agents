# 示例

本目录包含示例输出，展示如何协调代理机构的代理来解决现实世界的任务。

## 为什么存在这里

agency-agents 仓库定义了数十个专业代理，涵盖工程、设计、营销、产品、支持、空间计算和项目管理。但仅代理定义并不能展示当您在单一任务上**同时部署所有代理**时会发生什么。

这些示例回答了这个问题："当整个代理机构协作时，实际上是什么样子？"

## 内容

### [nexus-spatial-discovery.md](./nexus-spatial-discovery-zh.md)

**内容：** 一个完整的产品发现练习，8 个代理并行工作以评估软件机会并生成统一计划。

**场景：** Web 研究发现在 AI 代理编排和空间计算交叉点的机会。然后同时部署整个代理机构生成：

- 市场验证和竞争分析
- 技术架构（带有完整 SQL schema 的 8 服务系统设计）
- 品牌战略和视觉识别
- 上市和增长计划
- 客户支持运营蓝图
- UX 研究计划和角色旅程地图
- 35 周项目执行计划，包含 65 个冲刺票
- 空间界面架构规范

**使用的代理：**
| 代理 | 角色 |
|-------|------|
| 产品趋势研究员 | 市场验证、竞争格局 |
| 后端架构师 | 系统架构、数据模型、API 设计 |
| 品牌守护者 | 定位、视觉识别、命名 |
| 增长黑客 | GTM 战略、定价、发布计划 |
| 支持响应者 | 支持层级、入职、社区 |
| UX 研究员 | 角色、旅程地图、设计原则 |
| 项目牧羊人 | 阶段计划、冲刺、风险登记册 |
| XR 界面架构师 | 空间 UI 规范 |

**关键要点：** 所有 8 个代理并行运行，生成连贯的、相互引用的计划，没有协调开销。输出展示了代理机构从"找到机会"到"这是完整蓝图"的单次会话能力。

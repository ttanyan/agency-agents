# 多代理工作流：着陆页冲刺

> 使用 4 个代理在一天内发布转化优化的着陆页。

## 场景

您需要为新产品发布创建着陆页。它需要看起来很棒、转化访客，并在当天结束时上线。

## 代理团队

| 代理 | 在此工作流中的角色 |
|-------|---------------------|
| 内容创作者 | 撰写文案 |
| UI 设计师 | 设计布局和组件规格 |
| 前端开发者 | 构建它 |
| 增长黑客 | 优化转化 |

## 工作流程

### 上午：文案 + 设计（并行）

**步骤 1a — 激活内容创作者**

```
激活内容创作者。

为 "FlowSync" 撰写着陆页文案 -- 一个 API 集成平台
可在 5 分钟内连接任意两个 SaaS 工具。

目标受众：中型公司的开发者和产品技术经理。
语气：自信、简洁、略带俏皮。

需要的部分：
1. Hero（标题 + 副标题 + CTA）
2. 问题陈述（3 个痛点）
3. 工作原理（3 个步骤）
4. 社会证明（占位符推荐格式）
5. 定价（3 层：免费、专业、企业）
6. 最终 CTA

保持可扫描。没有废话。
```

**步骤 1b — 激活 UI 设计师（并行）**

```
激活 UI 设计师。

为 SaaS 着陆页设计规范。产品：FlowSync（API 集成平台）。
风格：干净、现代、深色模式选项。考虑 Linear 或 Vercel 美学。

交付：
1. 布局线框（部分顺序 + 间距）
2. 颜色调色板（主色、辅助色、强调色、背景色）
3. 排版（字体配对、标题大小、正文字体大小）
4. 组件规格：hero 部分、功能卡片、定价表、CTA 按钮
5. 响应式断点（移动端、平板、桌面）
```

### 中午：构建

**步骤 2 — 激活前端开发者**

```
激活前端开发者。

根据这些规格构建着陆页：

文案：[粘贴内容创作者输出]
设计：[粘贴 UI 设计师输出]

技术栈：HTML、Tailwind CSS、最少的原生 JS（不需要框架）。
要求：
- 响应式（移动优先）
- 快速（无重型资源，系统字体即可）
- 无障碍（适当的标题、alt 文本、焦点状态）
- 包含可用的电子邮件注册表单（action URL: /api/subscribe）

交付单个 index.html 文件，准备部署。
```

### 下午：优化

**步骤 3 — 激活增长黑客**

```
激活增长黑客。

审查此着陆页的转化优化：

[粘贴 HTML 或描述当前页面]

评估：
1. CTA 是否在首屏？
2. 价值主张是否在 5 秒内清晰？
3. 注册流程中是否有摩擦？
4. 您会首先运行哪些 A/B 测试？
5. SEO 基础：meta 标签、OG 标签、结构化数据

给我具体的更改，不是一般性建议。
```

## 时间线

| 时间 | 活动 | 代理 |
|------|----------|-------|
| 9:00 | 文案 + 设计启动（并行） | 内容创作者 + UI 设计师 |
| 11:00 | 构建开始 | 前端开发者 |
| 14:00 | 第一版就绪 | — |
| 14:30 | 转化审查 | 增长黑客 |
| 15:30 | 应用反馈 | 前端开发者 |
| 16:30 | 发布 | 部署到 Vercel/Netlify |

## 关键模式

1. **并行启动**：文案和设计同时发生，因为它们是独立的
2. **合并点**：前端开发者在开始之前需要两个输出
3. **反馈循环**：增长黑客审查，然后前端开发者应用更改
4. **时间盒**：每个步骤都有明确的时间盒以防止范围蔓延

# ⚡ NEXUS 快速入门指南

> **在 5 分钟内从零到编排的多代理管道。**

---

## 什么是 NEXUS？

**NEXUS**（Network of EXperts, Unified in Strategy- 专家网络，战略统一）将 The Agency 的 AI 专家转变为协调的管道。NEXUS 不是逐个激活代理并希望它们协同工作，而是准确定义每个步骤中谁做什么、何时做以及如何验证质量。

## 选择您的模式

| 我想要... | 使用 | 代理数量 | 时间 |
|-------------|-----|--------|------|
| 从零开始构建完整产品 | **NEXUS-Full** | 全部 | 12-24 周 |
| 构建功能或 MVP | **NEXUS-Sprint** | 15-25 | 2-6 周 |
| 执行特定任务（bug 修复、活动、审计） | **NEXUS-Micro** | 5-10 | 1-5 天 |

## 🚀 NEXUS-Full：启动完整项目

**复制此提示以激活完整管道：**

```
激活 Agents Orchestrator 为 NEXUS-Full 模式。

项目：[您的项目名称]
规格：[描述您的项目或链接到规格文档]

执行完整的 NEXUS 管道：
- 阶段 0：发现（趋势研究员、反馈综合师、UX 研究员、分析报告员、法律合规检查员、工具评估员）
- 阶段 1：战略（制片厂制片人、高级项目经理、冲刺优先级排序器、UX 架构师、品牌守护者、后端架构师、财务追踪器）
- 阶段 2：基础（DevOps 自动化器、前端开发者、后端架构师、UX 架构师、基础设施维护员）
- 阶段 3：构建（Dev↔QA 循环 — 所有工程 + 证据收集器）
- 阶段 4：加固（现实检验者、性能基准测试员、API 测试员、法律合规检查员）
- 阶段 5：发布（增长黑客、内容创作者、所有营销代理、DevOps 自动化器）
- 阶段 6：运营（分析报告员、基础设施维护员、支持响应者，持续进行）

每个阶段之间都有质量门。所有评估都需要证据。
每个任务最多 3 次重试，然后升级。
```

## 🎯 NEXUS-Micro：执行特定任务

**选择您的场景并复制提示：**

### 修复 Bug
```
激活后端架构师调查和修复 [BUG 描述]。
修复后，激活 API 测试员验证修复。
然后激活证据收集器确认没有视觉回归。
```

### 运行营销活动
```
激活社交媒体策略师作为 [活动描述] 的活动负责人。
团队：内容创作者、Twitter 互动者、Instagram 策展人、Reddit 社区建设者。
品牌守护者在发布前审查所有内容。
分析报告员每日追踪绩效。
增长黑客每周优化渠道。
```

### 进行合规审计
```
激活法律合规检查员进行全面合规审计。
范围：[GDPR / CCPA / HIPAA / 全部]
审计后，激活执行摘要生成器创建利益相关者报告。
```

### 调查性能问题
```
激活性能基准测试员诊断性能问题。
范围：[API 响应时间 / 页面加载 / 数据库查询 / 全部]
诊断后，激活基础设施维护员进行优化。
DevOps 自动化器部署任何基础设施更改。
```

### 市场研究
```
激活趋势研究员获取 [领域] 的市场情报。
交付物：竞争格局、市场规模、趋势预测。
研究后，激活执行摘要生成器创建执行简报。
```

### UX 改进
```
激活 UX 研究员识别 [功能/产品] 中的可用性问题。
研究后，激活 UX 架构师设计改进方案。
前端开发者实施更改。
证据收集器验证改进。
```

## 🔑 30 秒关键概念

1. **质量门** — 没有基于证据的批准，阶段不得推进
2. **Dev↔QA 循环** — 每个任务都要构建然后测试；PASS 才能继续，FAIL 则重试（最多 3 次）
3. **移交** — 代理之间的结构化上下文转移（永远不要冷启动）
4. **现实检验者** — 最终质量权威；默认为"NEEDS WORK"
5. **Agents Orchestrator** — 管理整个流程的管道控制器
6. **证据胜于声明** — 截图、测试结果和数据 -- 不是断言

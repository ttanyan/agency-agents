# 📋 NEXUS 交接模板

> NEXUS 管道中每种代理到代理交接的标准化模板。一致的交接可防止上下文丢失 — 这是多代理协调失败的 #1 原因。

---

## 1. 标准交接模板

用于任何代理到代理的工作转移。

```markdown
# NEXUS 交接文档

## 元数据
| 字段 | 值 |
|------|------|
| **发送方** | [代理名称] ([部门]) |
| **接收方** | [代理名称] ([部门]) |
| **阶段** | 阶段 [N] — [阶段名称] |
| **任务参考** | [来自冲刺优先排序器待办事项的任务 ID] |
| **优先级** | [关键 / 高 / 中 / 低] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 上下文
**项目**：[项目名称]
**当前状态**：[到目前为止已完成的工作 — 具体说明]
**相关文件**：
- [file/path/1] — [包含内容]
- [file/path/2] — [包含内容]
**依赖项**：[这项工作依赖于哪些已完成的内容]
**约束**：[技术、时间线或资源约束]

## 交付物请求
**需要什么**：[具体、可衡量的交付物描述]
**验收标准**：
- [ ] [标准 1 — 可衡量]
- [ ] [标准 2 — 可衡量]
- [ ] [标准 3 — 可衡量]
**参考材料**：[规格、设计、先前工作的链接]

## 质量期望
**必须通过**：[此交付物的具体质量标准]
**需要的证据**：[完成证明的样子]
**交接给下一个**：[谁接收输出以及他们需要什么格式]
```

## 2. QA 反馈循环 — 通过

当证据收集器或其他 QA 代理批准任务时使用。

```markdown
# NEXUS QA  verdict: 通过 ✅

## 任务
| 字段 | 值 |
|------|------|
| **任务 ID** | [ID] |
| **任务描述** | [描述] |
| **开发者代理** | [代理名称] |
| **QA 代理** | [代理名称] |
| **尝试次数** | [N] of 3 |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

##  verdict: 通过

## 证据
**截图**：
- 桌面 (1920x1080)：[文件名/路径]
- 平板 (768x1024)：[文件名/路径]
- 移动 (375x667)：[文件名/路径]

**功能验证**：
- [x] [验收标准 1] — 已验证
- [x] [验收标准 2] — 已验证
- [x] [验收标准 3] — 已验证

**品牌一致性**：已验证 — 颜色、排版、间距与设计系统匹配
**可访问性**：已验证 — 键盘导航、对比度、语义 HTML
**性能**：[测量的加载时间] — 在可接受范围内

## 备注
[任何观察、未来改进的小建议或积极评价]

## 下一步行动
→ 代理编排器：标记任务完成，推进到待办事项中的下一个任务
```

## 3. QA 反馈循环 — 失败

当证据收集器或其他 QA 代理拒绝任务时使用。

```markdown
# NEXUS QA  verdict: 失败 ❌

## 任务
| 字段 | 值 |
|------|------|
| **任务 ID** | [ID] |
| **任务描述** | [描述] |
| **开发者代理** | [代理名称] |
| **QA 代理** | [代理名称] |
| **尝试次数** | [N] of 3 |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

##  verdict: 失败

## 发现的问题

### 问题 1：[类别] — [严重程度：关键/高/中/低]
**描述**：[问题的准确描述]
**预期**：[根据验收标准应该发生的情况]
**实际**：[实际发生的情况]
**证据**：[截图文件名或测试输出]
**修复说明**：[具体、可操作的解决说明]
**要修改的文件**：[确切的文件路径]

### 问题 2：[类别] — [严重程度]
**描述**：[...]
**预期**：[...]
**实际**：[...]
**证据**：[...]
**修复说明**：[...]
**要修改的文件**：[...]

[继续列出所有发现的问题]

## 验收标准状态
- [x] [标准 1] — 通过
- [ ] [标准 2] — 失败 (见问题 1)
- [ ] [标准 3] — 失败 (见问题 2)

## 重试说明
**对于开发者代理**：
1. 仅修复上述列出的问题
2. 不要引入新功能或更改
3. 所有问题解决后重新提交 QA
4. 这是最多 3 次尝试中的第 [N] 次

**如果第 3 次尝试失败**：任务将升级到代理编排器
```

## 4. 升级报告

当任务超过 3 次重试尝试时使用。

```markdown
# NEXUS 升级报告 🚨

## 任务
| 字段 | 值 |
|------|------|
| **任务 ID** | [ID] |
| **任务描述** | [描述] |
| **开发者代理** | [代理名称] |
| **QA 代理** | [代理名称] |
| **已用尽尝试次数** | 3/3 |
| **升级至** | [代理编排器 / 工作室制作人] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 失败历史

### 尝试 1
- **发现的问题**：[摘要]
- **应用的修复**：[开发者更改了什么]
- **结果**：失败 — [为什么仍然失败]

### 尝试 2
- **发现的问题**：[摘要]
- **应用的修复**：[开发者更改了什么]
- **结果**：失败 — [为什么仍然失败]

### 尝试 3
- **发现的问题**：[摘要]
- **应用的修复**：[开发者更改了什么]
- **结果**：失败 — [为什么仍然失败]

## 根本原因分析
**任务持续失败的原因**：[对潜在问题的分析]
**系统性问题**：[这是一次性问题还是模式？]
**复杂性评估**：[任务范围是否适当？]

## 推荐解决方案
- [ ] **重新分配**给不同的开发者代理 ([推荐代理])
- [ ] **分解**为更小的子任务 ([提议的分解])
- [ ] **修改方法** — 需要架构/设计变更
- [ ] **接受**当前状态，记录限制
- [ ] **推迟**到未来冲刺

## 影响评估
**阻塞**：[此任务阻塞了哪些其他任务]
**时间线影响**：[这如何影响整体时间表]
**质量影响**：[如果我们接受当前状态，存在哪些质量妥协]

## 需要决策
**决策者**：[代理编排器 / 工作室制作人]
**截止日期**：[需要决策以避免进一步延迟的时间]
```

## 5. 阶段门控交接

在 NEXUS 阶段之间过渡时使用。

```markdown
# NEXUS 阶段门控交接

## 过渡
| 字段 | 值 |
|------|------|
| **从阶段** | 阶段 [N] — [名称] |
| **到阶段** | 阶段 [N+1] — [名称] |
| **门控守护者** | [代理名称(s)] |
| **门控结果** | [通过 / 失败] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 门控标准结果
| # | 标准 | 阈值 | 结果 | 证据 |
|---|-------|-------|------|------|
| 1 | [标准] | [阈值] | ✅ 通过 / ❌ 失败 | [证据参考] |
| 2 | [标准] | [阈值] | ✅ 通过 / ❌ 失败 | [证据参考] |
| 3 | [标准] | [阈值] | ✅ 通过 / ❌ 失败 | [证据参考] |

## 携带到下一阶段的文档
1. [文档名称] — [下一阶段的用途]
2. [文档名称] — [下一阶段的用途]
3. [文档名称] — [下一阶段的用途]

## 下一阶段的关键约束

## 下一阶段的代理激活

## 携带到下一阶段的风险
| 风险 | 严重程度 | 缓解措施 | 所有者 |
|------|----------|----------|--------|
| [风险] | [P0-P3] | [缓解计划] | [代理] |
```

## 7. 事件交接

在事件响应期间使用。

```markdown
# NEXUS 事件交接

## 事件
| 字段 | 值 |
|------|------|
| **严重程度** | [P0 / P1 / P2 / P3] |
| **检测者** | [代理或系统] |
| **检测时间** | [时间戳] |
| **分配给** | [代理名称] |
| **状态** | [调查中 / 缓解中 / 已解决 / 事后分析] |

## 描述
**发生了什么**：[事件的清晰描述]
**影响**：[谁/什么受到影响以及严重程度如何]
**时间线**：
- [HH:MM] — [事件]
- [HH:MM] — [事件]
- [HH:MM] — [事件]

## 当前状态
**受影响的系统**：[列表]
**可用的解决方法**：[是/否 — 如果是，描述]
**估计解决时间**：[时间估计]

## 已采取的行动
1. [采取的行动和结果]
2. [采取的行动和结果]

## 交接上下文
**对于下一个响应者**：
- [已尝试的内容]
- [尚未尝试的内容]
- [疑似根本原因]
- [要检查的相关日志/指标]

## 利益相关者沟通
**上次发送的更新**：[时间戳]
**下次更新截止**：[时间戳]
**沟通渠道**：[更新发布的位置]
```

## 使用指南

| 情况 | 使用的模板 |
|------|------------|
| 向另一个代理分配工作 | 标准交接 (#1) |
| QA 批准任务 | QA 通过 (#2) |
| QA 拒绝任务 | QA 失败 (#3) |
| 任务超过 3 次重试 | 升级报告 (#4) |
| 在阶段之间移动 | 阶段门控交接 (#5) |
| 冲刺结束 | 冲刺交接 (#6) |
| 系统事件 | 事件交接 (#7) |

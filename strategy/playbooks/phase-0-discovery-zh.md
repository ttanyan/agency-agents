# 🔍 第 0 阶段手册 — 情报与发现

> **持续时间**：3-7 天 | **代理**：6 | **门控守护者**：执行摘要生成器

---

## 目标

在投入资源之前验证机会。在理解问题、市场和监管环境之前不要开始构建。

## 前提条件

- [ ] 项目简介或初始概念存在
- [ ] 已确定利益相关者发起人
- [ ] 发现阶段预算已批准

## 代理激活序列

### 第一波：并行启动（第 1 天）

#### 🔍 趋势研究员 — 市场情报负责人
```
激活趋势研究员进行 [项目领域] 的市场情报分析。

所需交付物：
1. 竞争格局分析（直接 + 间接竞争对手）
2. 市场规模：TAM、SAM、SOM 及其方法论
3. 趋势生命周期映射：该市场在采用曲线中的位置？
4. 3-6 个月趋势预测，带置信区间
5. 该领域的投资和资金趋势

来源：至少 15 个独特的、经过验证的来源
格式：带执行摘要的战略报告
时间线：3 天
```

#### 💬 反馈合成器 — 用户需求分析
```
激活反馈合成器进行 [项目领域] 的用户需求分析。

所需交付物：
1. 多渠道反馈收集计划（调查、访谈、评论、社交）
2. 现有用户接触点的情感分析
3. 痛点识别和优先级排序（RICE 评分）
4. 带有商业价值估算的功能请求分析
5. 来自反馈模式的流失风险指标

格式：带优先级矩阵的合成反馈报告
时间线：3 天
```

#### 🔍 UX 研究员 — 用户行为分析
```
激活 UX 研究员进行 [项目领域] 的用户行为分析。

所需交付物：
1. 用户访谈计划（5-10 个目标用户）
2. 人物角色开发（3-5 个主要人物角色）
3. 主要用户流程的旅程映射
4. 竞争对手产品的可用性启发式评估
5. 带有统计验证的行为洞察

格式：带有人物角色和旅程地图的研究发现报告
时间线：5 天
```

### 第二波：并行启动（第 1 天，独立于第一波）

#### 📊 分析报告器 — 数据景观评估
```
激活分析报告器进行 [项目领域] 的数据景观评估。

所需交付物：
1. 现有数据源审计（有哪些数据可用？）
2. 信号识别（我们可以测量什么？）
3. 基线指标建立
4. 数据质量评估，带完整性评分
5. 分析基础设施建议

格式：带有信号图的数据审计报告
时间线：2 天
```

#### ⚖️ 法律合规检查器 — 监管扫描
```
激活法律合规检查器进行 [项目领域] 的监管扫描。

所需交付物：
1. 适用的监管框架（GDPR、CCPA、HIPAA 等）
2. 数据处理要求和约束
3. 目标市场的司法管辖区映射
4. 合规风险评估，带严重程度评级
5. 阻塞性与可管理的合规问题

格式：合规要求矩阵
时间线：3 天
```

#### 🛠️ 工具评估器 — 技术景观
```
激活工具评估器进行 [项目领域] 的技术景观评估。

所需交付物：
1. 问题域的技术栈评估
2. 关键组件的构建 vs 购买分析
3. 与现有系统的集成可行性
4. 开源 vs 商业评估
5. 技术风险评估

格式：带推荐矩阵的技术栈评估
时间线：2 天
```

## 收敛点（第 5-7 天）

所有六个代理交付他们的报告。执行摘要生成器综合：

```
激活执行摘要生成器综合第 0 阶段的发现。

输入文档：
1. 趋势研究员 → 市场分析报告
2. 反馈合成器 → 合成反馈报告
3. UX 研究员 → 研究发现报告
4. 分析报告器 → 数据审计报告
5. 法律合规检查器 → 合规要求矩阵
6. 工具评估器 → 技术栈评估

输出：执行摘要（≤500 字，SCQA 格式）
需要决策：GO / NO-GO / PIVOT
包含：量化的市场机会、验证的用户需求、监管路径、技术可行性
```

## 质量门控清单

| # | 标准 | 证据来源 | 状态 |
|---|-------|----------|------|
| 1 | 市场机会已验证，TAM > 最小可行阈值 | 趋势研究员报告 | ☐ |
| 2 | ≥3 个已验证的用户痛点，有支持数据 | 反馈合成器 + UX 研究员 | ☐ |
| 3 | 未识别出阻塞性合规问题 | 法律合规检查器矩阵 | ☐ |
| 4 | 已识别关键指标和数据源 | 分析报告器审计 | ☐ |
| 5 | 技术栈可行且已评估 | 工具评估器评估 | ☐ |
| 6 | 交付了带有 GO/NO-GO 建议的执行摘要 | 执行摘要生成器 | ☐ |

## 门控决策

- **GO**：进入第 1 阶段 — 策略与架构
- **NO-GO**：存档发现，记录学习，重定向资源
- **PIVOT**：根据发现修改范围/方向，重新运行有针对性的发现

## 交接给第 1 阶段

```markdown
## 第 0 阶段 → 第 1 阶段交接包

### 携带到下一阶段的文档：
1. 市场分析报告（趋势研究员）
2. 合成反馈报告（反馈合成器）
3. 用户人物角色和旅程地图（UX 研究员）
4. 数据审计报告（分析报告器）
5. 合规要求矩阵（法律合规检查器）
6. 技术栈评估（工具评估器）
7. 带有 GO 决策的执行摘要（执行摘要生成器）

### 已识别的关键约束：
- [来自法律合规检查器的监管约束]
- [来自工具评估器的技术约束]
- [来自趋势研究员的市场时机约束]

### 优先用户需求（供冲刺优先排序器使用）：
1. [痛点 1 — 来自反馈合成器]
2. [痛点 2 — 来自 UX 研究员]
3. [痛点 3 — 来自反馈合成器]
```

# 🏗️ 第 1 阶段手册 — 策略与架构

> **持续时间**：5-10 天 | **代理**：8 | **门控守护者**：工作室制作人 + 现实检查器

---

## 目标

在编写一行代码之前，定义我们要构建什么、如何构建以及成功的样子。每个架构决策都有记录。每个功能都有优先级。每一分钱都有说明。

## 前提条件

- [ ] 第 0 阶段质量门控通过（GO 决策）
- [ ] 收到第 0 阶段交接包
- [ ] 利益相关者对项目范围达成一致

## 代理激活序列

### 步骤 1：战略框架（第 1-3 天，并行）

#### 🎬 工作室制作人 — 战略组合对齐
```
激活工作室制作人进行 [项目] 的战略组合对齐。

输入：第 0 阶段执行摘要 + 市场分析报告
所需交付物：
1. 带有项目定位的战略组合计划
2. 愿景、目标和 ROI 目标
3. 资源分配策略
4. 风险/回报评估
5. 成功标准和里程碑定义

对齐：组织战略目标
格式：战略组合计划模板
时间线：3 天
```

#### 🎭 品牌守护者 — 品牌标识系统
```
激活品牌守护者进行 [项目] 的品牌标识开发。

输入：第 0 阶段 UX 研究（人物角色、旅程地图）
所需交付物：
1. 品牌基础（目的、愿景、使命、价值观、个性）
2. 视觉标识系统（颜色、排版、间距作为 CSS 变量）
3. 品牌声音和消息架构
4. 标志系统规格（如果是新品牌）
5. 品牌使用指南

格式：品牌标识系统文档
时间线：3 天
```

#### 💰 财务跟踪器 — 预算和资源规划
```
激活财务跟踪器进行 [项目] 的财务规划。

输入：工作室制作人战略计划 + 第 0 阶段技术栈评估
所需交付物：
1. 带有类别细分的综合项目预算
2. 资源成本预测（代理、基础设施、工具）
3. 带有盈亏平衡分析的 ROI 模型
4. 现金流时间线
5. 带有应急储备的财务风险评估

格式：带有 ROI 预测的财务计划
时间线：2 天
```

### 步骤 2：技术架构（第 3-7 天，并行，在步骤 1 输出可用后）

#### 🏛️ UX 架构师 — 技术架构 + UX 基础
```
激活 UX 架构师进行 [项目] 的技术架构。

输入：品牌守护者视觉标识 + 第 0 阶段 UX 研究
所需交付物：
1. CSS 设计系统（变量、令牌、比例）
2. 布局框架（网格/弹性盒模式、响应式断点）
3. 组件架构（命名约定、层次结构）
4. 信息架构（页面流程、内容层次结构）
5. 主题系统（浅色/深色/系统切换）
6. 可访问性基础（WCAG 2.1 AA 基线）

要创建的文件：
- css/design-system.css
- css/layout.css
- css/components.css
- docs/ux-architecture.md

格式：开发者就绪的基础包
时间线：4 天
```

#### 🏗️ 后端架构师 — 系统架构
```
激活后端架构师进行 [项目] 的系统架构。

输入：第 0 阶段技术栈评估 + 合规要求
所需交付物：
1. 系统架构规格
   - 架构模式（微服务/单体/无服务器/混合）
   - 通信模式（REST/GraphQL/gRPC/事件驱动）
   - 数据模式（CQRS/事件溯源/CRUD）
2. 带索引策略的数据库模式设计
3. 带版本控制的 API 设计规格
4. 身份验证和授权架构
5. 安全架构（深度防御）
6. 可扩展性计划（水平扩展策略）

格式：系统架构规格
时间线：4 天
```

#### 🤖 AI 工程师 — ML 架构（如适用）
```
激活 AI 工程师进行 [项目] 的 ML 系统架构。

输入：后端架构师系统架构 + 第 0 阶段数据审计
所需交付物：
1. ML 系统设计
   - 模型选择和训练策略
   - 数据管道架构
   - 推理策略（实时/批处理/边缘）
2. AI 伦理和安全框架
3. 模型监控和再训练计划
4. 与主应用程序的集成点
5. ML 基础设施的成本预测

条件：仅当项目包含 AI/ML 功能时激活
格式：ML 系统设计文档
时间线：3 天
```

#### 👔 高级项目经理 — 规格到任务转换
```
激活高级项目经理进行 [项目] 的任务列表创建。

输入：所有第 0 阶段文档 + 架构规格（如可用）
所需交付物：
1. 综合任务列表
   - 引用规格中的确切要求（无奢侈功能）
   - 每个任务都有明确的验收标准
   - 任务之间的依赖关系映射
   - 工作量估算（故事点或小时）
2. 工作分解结构
3. 关键路径识别
4. 实施风险登记册

规则：
- 不要添加规格中没有的功能
- 引用要求中的确切文本
- 对工作量估算要现实

格式：带验收标准的任务列表
时间线：3 天
```

### 步骤 3：优先级排序（第 7-10 天，顺序，在步骤 2 之后）

#### 🎯 冲刺优先排序器 — 功能优先级
```
激活冲刺优先排序器进行 [项目] 的待办事项优先级排序。

输入：
- 高级项目经理 → 任务列表
- 后端架构师 → 系统架构
- UX 架构师 → UX 架构
- 财务跟踪器 → 预算框架
- 工作室制作人 → 战略计划

所需交付物：
1. RICE 评分的待办事项（Reach、Impact、Confidence、Effort）
2. 基于速度估计的冲刺分配
3. 带有关键路径的依赖关系图
4. MoSCoW 分类（Must/Should/Could/Won't）
5. 带有里程碑映射的发布计划

验证：工作室制作人确认战略对齐
格式：优先排序的冲刺计划
时间线：2 天
```

## 质量门控清单

| # | 标准 | 证据来源 | 状态 |
|---|-------|----------|------|
| 1 | 架构覆盖 100% 的规格要求 | 高级 PM 任务列表与架构交叉引用 | ☐ |
| 2 | 品牌系统完整（标志、颜色、排版、声音） | 品牌守护者交付物 | ☐ |
| 3 | 所有技术组件都有实现路径 | 后端架构师 + UX 架构师规格 | ☐ |
| 4 | 预算已批准且在约束范围内 | 财务跟踪器计划 | ☐ |
| 5 | 冲刺计划基于速度且现实 | 冲刺优先排序器待办事项 | ☐ |
| 6 | 安全架构已定义 | 后端架构师安全规格 | ☐ |
| 7 | 合规要求已集成到架构中 | 法律要求映射到技术决策 | ☐ |

## 门控决策

**需要双重签字**：工作室制作人（战略）+ 现实检查器（技术）

- **批准**：携带完整架构包进入第 2 阶段
- **修订**：特定项目需要返工（返回相关步骤）
- **重构**：基本架构问题（重新开始第 1 阶段）

## 交接给第 2 阶段

```markdown
## 第 1 阶段 → 第 2 阶段交接包

### 架构包：
1. 战略组合计划（工作室制作人）
2. 品牌标识系统（品牌守护者）
3. 财务计划（财务跟踪器）
4. CSS 设计系统 + UX 架构（UX 架构师）
5. 系统架构规格（后端架构师）
6. ML 系统设计（AI 工程师 — 如适用）
7. 综合任务列表（高级项目经理）
8. 优先排序的冲刺计划（冲刺优先排序器）

### 给 DevOps 自动化师：
- 后端架构师的部署架构
- 系统架构中的环境要求
- 基础设施需求中的监控要求

### 给前端开发者：
- UX 架构师的 CSS 设计系统
- 品牌守护者的品牌标识
- UX 架构师的组件架构
- 后端架构师的 API 规格

### 给后端架构师（继续）：
- 数据库模式准备就绪，可部署
- API 脚手架准备就绪，可实现
- 认证系统架构已定义
```

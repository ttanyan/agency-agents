# ⚙️ 第 2 阶段手册 — 基础与脚手架

> **持续时间**：3-5 天 | **代理**：6 | **门控守护者**：DevOps 自动化师 + 证据收集器

---

## 目标

构建所有后续工作所依赖的技术和运营基础。在添加肌肉之前先搭建骨架。在此阶段之后，每个开发者都有一个工作环境、一个可部署的管道和一个可构建的设计系统。

## 前提条件

- [ ] 第 1 阶段质量门控通过（架构包已批准）
- [ ] 收到第 1 阶段交接包
- [ ] 所有架构文档已完成

## 代理激活序列

### 工作流 A：基础设施（第 1-3 天，并行）

#### 🚀 DevOps 自动化师 — CI/CD 管道 + 基础设施
```
激活 DevOps 自动化师进行 [项目] 的基础设施设置。

输入：后端架构师系统架构 + 部署要求
所需交付物：
1. CI/CD 管道（GitHub Actions / GitLab CI）
   - 安全扫描阶段
   - 自动化测试阶段
   - 构建和容器化阶段
   - 部署阶段（蓝绿或金丝雀）
   - 自动回滚能力
2. 基础设施即代码
   - 环境配置（开发、暂存、生产）
   - 容器编排设置
   - 网络和安全配置
3. 环境配置
   - 密钥管理
   - 环境变量管理
   - 多环境一致性

要创建的文件：
- .github/workflows/ci-cd.yml（或等效文件）
- infrastructure/（Terraform/CDK 模板）
- docker-compose.yml
- Dockerfile(s)

格式：带有 IaC 模板的工作 CI/CD 管道
时间线：3 天
```

#### 🏗️ 基础设施维护者 — 云基础设施 + 监控
```
激活基础设施维护者进行 [项目] 的监控设置。

输入：DevOps 自动化师基础设施 + 后端架构师架构
所需交付物：
1. 云资源配置
   - 计算、存储、网络资源
   - 自动扩展配置
   - 负载均衡器设置
2. 监控栈
   - 应用指标（Prometheus/DataDog）
   - 基础设施指标
   - 自定义仪表板（Grafana）
3. 日志和警报
   - 集中式日志聚合
   - 关键阈值的警报规则
   - 值班通知设置
4. 安全加固
   - 防火墙规则
   - SSL/TLS 配置
   - 访问控制策略

格式：带有仪表板访问的基础设施就绪报告
时间线：3 天
```

#### ⚙️ 工作室运营 — 流程设置
```
激活工作室运营进行 [项目] 的流程设置。

输入：冲刺优先排序器计划 + 项目牧羊人协调需求
所需交付物：
1. Git 工作流
   - 分支策略（GitFlow / 主干基于）
   - PR 审查流程
   - 合并策略
2. 沟通渠道
   - 团队渠道设置
   - 通知路由
   - 状态更新节奏
3. 文档模板
   - PR 模板
   - 问题模板
   - 决策日志模板
4. 协作工具
   - 项目看板设置
   - 冲刺跟踪配置

格式：运营手册
时间线：2 天
```

### 工作流 B：应用基础（第 1-4 天，并行）

#### 🎨 前端开发者 — 项目脚手架 + 组件库
```
激活前端开发者进行 [项目] 的项目脚手架。

输入：UX 架构师 CSS 设计系统 + 品牌守护者标识
所需交付物：
1. 项目脚手架
   - 框架设置（React/Vue/Angular 按架构）
   - TypeScript 配置
   - 构建工具（Vite/Webpack/Next.js）
   - 测试框架（Jest/Vitest + Testing Library）
2. 设计系统实现
   - 来自 UX 架构师的 CSS 设计令牌
   - 基础组件库（按钮、输入、卡片、布局）
   - 主题系统（浅色/深色/系统切换）
   - 响应式工具
3. 应用外壳
   - 路由设置
   - 布局组件（头部、底部、侧边栏）
   - 错误边界实现
   - 加载状态

要创建的文件：
- src/（应用源代码）
- src/components/（组件库）
- src/styles/（设计令牌）
- src/layouts/（布局组件）

格式：带有组件库的工作应用骨架
时间线：3 天
```

#### 🏗️ 后端架构师 — 数据库 + API 基础
```
激活后端架构师进行 [项目] 的 API 基础。

输入：系统架构规格 + 数据库模式设计
所需交付物：
1. 数据库设置
   - 模式部署（迁移）
   - 索引创建
   - 开发种子数据
   - 连接池配置
2. API 脚手架
   - 框架设置（Express/FastAPI/etc.）
   - 路由结构匹配架构
   - 中间件栈（认证、验证、错误处理、CORS）
   - 健康检查端点
3. 认证系统
   - 认证提供者集成
   - JWT/会话管理
   - 基于角色的访问控制脚手架
4. 服务通信
   - API 版本控制设置
   - 请求/响应序列化
   - 错误响应标准化

要创建的文件：
- api/ 或 server/（后端源代码）
- migrations/（数据库迁移）
- docs/api-spec.yaml（OpenAPI 规格）

格式：带有数据库和认证的工作 API 脚手架
时间线：4 天
```

#### 🏛️ UX 架构师 — CSS 系统实现
```
激活 UX 架构师进行 [项目] 的 CSS 系统实现。

输入：品牌守护者标识 + 自己的第 1 阶段 CSS 设计系统规格
所需交付物：
1. 设计令牌实现
   - CSS 自定义属性（颜色、排版、间距）
   - 带有语义命名的品牌调色板
   - 带有响应式调整的排版比例
2. 布局系统
   - 容器系统（响应式断点）
   - 网格模式（2 列、3 列、侧边栏）
   - 弹性盒工具
3. 主题系统
   - 浅色主题变量
   - 深色主题变量
   - 系统偏好检测
   - 主题切换组件
   - 主题之间的平滑过渡

要创建/更新的文件：
- css/design-system.css（或框架中的等效文件）
- css/layout.css
- css/components.css
- js/theme-manager.js

格式：带有主题切换的已实现 CSS 设计系统
时间线：2 天
```

## 门控决策

**需要双重签字**：DevOps 自动化师（基础设施）+ 证据收集器（视觉）

- **通过**：带有完整 DevOps 管道的工作骨架 → 激活第 3 阶段
- **失败**：特定基础设施或应用问题 → 修复并重新验证

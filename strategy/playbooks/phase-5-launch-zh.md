# 🚀 第 5 阶段手册 — 发布与增长

> **持续时间**：2-4 周（T-7 到 T+14） | **代理**：12 | **门控守护者**：工作室制作人 + 分析报告器

---

## 目标

同时协调所有渠道的上市执行。发布时的最大影响。每个营销代理协同工作，同时工程确保稳定性。

## 前提条件

- [ ] 第 4 阶段质量门控通过（现实检查器就绪 verdict）
- [ ] 收到第 4 阶段交接包
- [ ] 生产部署计划已批准
- [ ] 营销内容管道就绪（来自第 3 阶段轨道 B）

## 发布时间线

### T-7：发布前一周

#### 内容与活动准备（并行）

```
激活内容创建者：
- 完成所有发布内容（博客文章、登陆页面、电子邮件序列）
- 在发布平台中排队内容
- 准备针对预期问题的响应模板
- 创建发布日实时内容计划

激活社交媒体策略师：
- 完成跨平台活动资产
- 安排发布前预告内容
- 协调影响者合作关系
- 准备平台特定的内容变体

激活增长黑客：
- 配置病毒机制（推荐代码、分享激励）
- 配置增长实验跟踪
- 设置漏斗分析
- 准备获取渠道预算

激活应用商店优化师（如果是移动应用）：
- 完成商店列表（标题、描述、关键词、截图）
- 提交应用审核（如适用）
- 准备发布日 ASO 调整
- 配置应用内评价提示
```

#### 技术准备（并行）

```
激活 DevOps 自动化师：
- 准备蓝绿部署
- 验证回滚程序
- 配置功能标志以进行渐进式发布
- 端到端测试部署管道

激活基础设施维护者：
- 配置自动扩展以应对 10 倍预期流量
- 验证监控和警报阈值
- 测试灾难恢复程序
- 准备事件响应运行手册

- 向所有代理分发发布清单
- 确认所有依赖项已解决
- 设置发布日沟通渠道
- 向利益相关者简报发布计划
```

### T-0：发布日

#### 第 0 小时：部署

```
激活 DevOps 自动化师：
1. 执行蓝绿部署到生产环境
2. 运行所有服务的健康检查
3. 验证数据库迁移完成
4. 确认所有端点响应
5. 将流量切换到新部署
6. 监控错误率 15 分钟
7. 确认：部署成功或回滚

激活基础设施维护者：
1. 实时监控所有系统指标
2. 观察流量峰值和扩展事件
3. 跟踪错误率和响应时间
4. 对任何阈值突破发出警报
5. 确认：系统稳定
```

## 门控决策

**双重签字**：工作室制作人（战略）+ 分析报告器（数据）

- **稳定**：产品已发布，系统稳定，增长活跃 → 激活第 6 阶段
- **关键**：需要立即工程响应的重大问题 → 热修复周期
- **回滚**：基本问题 → 恢复部署，返回第 4 阶段

# 🏢 运行手册：企业功能开发

> **模式**：NEXUS-Sprint | **持续时间**：6-12 周 | **代理**：20-30

---

## 场景

你正在为现有企业产品添加一个主要功能。合规、安全和质量门控是不可协商的。多个利益相关者需要对齐。该功能必须与现有系统无缝集成。

## 执行计划

### 阶段 1：需求与架构（第 1-2 周）

```
第 1 周：利益相关者对齐
├── 项目牧羊人 → 利益相关者分析 + 沟通计划
├── UX 研究员 → 功能需求的用户研究
├── 法律合规检查器 → 合规要求扫描
├── 高级项目经理 → 规格到任务转换
└── 财务跟踪器 → 预算框架

第 2 周：技术架构
├── UX 架构师 → UX 基础 + 组件架构
├── 后端架构师 → 系统架构 + 集成计划
├── UI 设计师 → 组件设计 + 设计系统更新
├── 冲刺优先排序器 → RICE 评分的待办事项
├── 品牌守护者 → 品牌影响评估
└── 质量门控：架构审查（项目牧羊人 + 现实检查器）
```

### 阶段 2：基础（第 3 周）

```
├── DevOps 自动化师 → 功能分支管道 + 功能标志
├── 前端开发者 → 组件脚手架
├── 后端架构师 → API 脚手架 + 数据库迁移
├── 基础设施维护者 → 暂存环境设置
└── 质量门控：基础验证（证据收集器）
```

### 阶段 3：构建（第 4-9 周）

```
冲刺 1-3（第 4-9 周）：
├── 代理编排器 → Dev↔QA 循环管理
├── 前端开发者 → UI 实现（逐个任务）
├── 后端架构师 → API 实现（逐个任务）
├── 高级开发者 → 复杂/高级功能
├── 证据收集器 → 每个任务的 QA（截图）
├── API 测试器 → 每个 API 任务的端点验证
├── 实验跟踪器 → 关键功能的 A/B 测试设置
│
├── 每两周：
│   ├── 项目牧羊人 → 利益相关者状态更新
│   ├── 执行摘要生成器 → 执行简报
│   └── 财务跟踪器 → 预算跟踪
│
└── 带有利益相关者演示的冲刺评审
```

### 阶段 4：加固（第 10-11 周）

```
第 10 周：证据收集
├── 证据收集器 → 完整截图套件
├── API 测试器 → 完整回归套件
├── 性能基准测试器 → 10 倍流量的负载测试
├── 法律合规检查器 → 最终合规审计
├── 测试结果分析器 → 质量指标仪表板
└── 基础设施维护者 → 生产就绪性

第 11 周：最终判断
├── 现实检查器 → 集成测试（默认：需要工作）
├── 如有需要的修复周期（2-3 天）
├── 重新验证
└── 执行摘要生成器 → 进入/不进入建议
```

### 阶段 5：推出（第 12 周）

```
├── DevOps 自动化师 → 金丝雀部署（5% → 25% → 100%）
├── 基础设施维护者 → 实时监控
├── 分析报告器 → 功能采用跟踪
├── 支持响应者 → 新功能的用户支持
├── 反馈合成器 → 早期反馈收集
└── 执行摘要生成器 → 发布报告
```

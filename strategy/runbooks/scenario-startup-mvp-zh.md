# 🚀 运行手册：创业公司MVP构建

> **模式**：NEXUS-Sprint | **持续时间**：4-6周 | **代理**：18-22个

---

## 场景

你正在构建一个创业公司MVP（最小可行产品）——一个需要快速验证产品市场契合度的新产品。速度很重要，但质量也同样重要。你需要在4-6周内从想法到有真实用户的上线产品。

## 代理名单

### 核心团队（始终活跃）
| 代理 | 角色 |
|-------|------|
| 代理编排器 | 管道控制器 |
| 高级项目经理 | 规范到任务转换 |
| Sprint 优先级排序器 | 待办事项管理 |
| UX 架构师 | 技术基础 |
| 前端开发者 | UI 实现 |
| 后端架构师 | API 和数据库 |
| DevOps 自动化专家 | CI/CD 和部署 |
| 证据收集器 | 每个任务的 QA |
| 现实检查器 | 最终质量门 |

### 增长团队（第3周+激活）
| 代理 | 角色 |
|-------|------|
| 增长黑客 | 获取策略 |
| 内容创建者 | 启动内容 |
| 社交媒体策略师 | 社交活动 |

### 支持团队（按需）
| 代理 | 角色 |
|-------|------|
| 品牌守护者 | 品牌标识 |
| 分析报告员 | 指标和仪表板 |
| 快速原型师 | 快速验证实验 |
| AI 工程师 | 如果产品包含 AI 功能 |
| 性能基准测试员 | 启动前负载测试 |
| 基础设施维护者 | 生产设置 |

## 每周执行

### 第1周：发现 + 架构（压缩的第0阶段 + 第1阶段）

```
第1-2天：压缩发现
├── 趋势研究员 → 快速竞争扫描（1天，非完整报告）
├── UX 架构师 → 关键用户流程线框图
└── 高级项目经理 → 将规范转换为任务列表

第3-4天：架构
├── UX 架构师 → CSS 设计系统 + 组件架构
├── 后端架构师 → 系统架构 + 数据库架构
├── 品牌守护者 → 快速品牌基础（颜色，排版，声音）
└── Sprint 优先级排序器 → RICE 评分的待办事项 + 冲刺计划

第5天：基础设置
├── DevOps 自动化专家 → CI/CD 管道 + 环境
├── 前端开发者 → 项目脚手架
├── 后端架构师 → 数据库 + API 脚手架
└── 质量门：架构包已批准
```

### 第2-3周：核心构建（第2阶段 + 第3阶段）

```
冲刺 1（第2周）：
├── 代理编排器管理开发↔QA 循环
├── 前端开发者 → 核心 UI（认证，主视图，导航）
├── 后端架构师 → 核心 API（认证，CRUD，业务逻辑）
├── 证据收集器 → 每个完成任务的 QA
├── AI 工程师 → 如适用的 ML 功能
└── 周末进行冲刺审查

冲刺 2（第3周）：
├── 继续剩余功能的开发↔QA 循环
├── 增长黑客 → 设计病毒机制 + 推荐系统
├── 内容创建者 → 开始启动内容创建
├── 分析报告员 → 设置跟踪和仪表板
└── 周末进行冲刺审查
```

### 第4周：打磨 + 加固（第4阶段）

```
第1-2天：质量冲刺
├── 证据收集器 → 完整屏幕截图套件
├── 性能基准测试员 → 负载测试
├── 前端开发者 → 修复 QA 问题
├── 后端架构师 → 修复 API 问题
└── 品牌守护者 → 品牌一致性审计

第3-4天：现实检查
├── 现实检查器 → 最终集成测试
├── 基础设施维护者 → 生产准备
└── DevOps 自动化专家 → 生产部署准备

第5天：门决策
├── 现实检查器裁决
├── 如果需要工作：快速修复周期（2-3天）
├── 如果准备就绪：继续启动
└── 执行摘要生成器 → 利益相关者简报
```

### 第5-6周：启动 + 增长（第5阶段）

```
第5周：启动
├── DevOps 自动化专家 → 生产部署
├── 增长黑客 → 激活获取渠道
├── 内容创建者 → 发布启动内容
├── 社交媒体策略师 → 跨平台活动
├── 分析报告员 → 实时监控
└── 支持响应者 → 用户支持激活

第6周：优化
├── 增长黑客 → 分析和优化渠道
├── 反馈合成器 → 收集早期用户反馈
├── 实验跟踪器 → 启动 A/B 测试
├── 分析报告员 → 第1周分析
└── Sprint 优先级排序器 → 计划迭代冲刺
```

## 关键决策

| 决策点 | 时间 | 决策者 |
|---------------|------|-------------|
| 概念的通过/不通过 | 第2天结束 | 工作室制作人 |
| 架构批准 | 第4天结束 | 高级项目经理 |
| MVP 的功能范围 | 冲刺规划 | Sprint 优先级排序器 |
| 生产准备就绪 | 第4周第5天 | 现实检查器 |
| 启动时间 | 现实检查器就绪后 | 工作室制作人 |

## 成功标准

| 指标 | 目标 |
|--------|--------|
| 上线产品时间 | ≤ 6周 |
| 核心功能完成 | MVP 范围的 100% |
| 首次用户入职 | 启动后 48 小时内 |
| 系统正常运行时间 | 第一周 > 99% |
| 收集的用户反馈 | 前 2 周 ≥ 50 个响应 |

## 常见陷阱与缓解措施

| 陷阱 | 缓解措施 |
|---------|-----------|
| 构建期间的范围蔓延 | Sprint 优先级排序器强制执行 MoSCoW — "不会" 意味着不会 |
| 为规模过度工程 | 快速原型师思维 — 先验证，后扩展 |
| 为了速度跳过 QA | 证据收集器对每个任务运行 — 无例外 |
| 无监控启动 | 基础设施维护者在第 1 周设置监控 |
| 无反馈机制 | 分析 + 反馈收集内置在冲刺 1 中 |
